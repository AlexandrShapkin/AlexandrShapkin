Инженер сопровождения и начинающий DevOps-инженер. Работаю с Linux, автоматизацией, контейнерами и эксплуатацией информационных систем.

Нахожусь в Барнауле и рассматриваю удалённую полную занятость в направлениях DevOps, Linux/system administration, infrastructure engineering и технического сопровождения L2/L3.

- **Linux и сети:** Linux, systemd, Bash, TCP/IP, DNS, Nginx, VPN
- **Контейнеры и оркестрация:** Docker, Docker Compose, Kubernetes, Helm
- **Автоматизация и IaC:** Ansible, Terraform, Python, Make
- **CI/CD и GitOps:** Git, GitLab CI, GitHub Actions, Argo CD
- **Наблюдаемость:** Prometheus, Grafana, Loki, Promtail
- **Данные:** Oracle Database, PostgreSQL, SQL, Redis, MongoDB
- **Виртуализация:** Proxmox VE, cloud-init

## Избранные проекты

### [homelab-infra](https://github.com/AlexandrShapkin/homelab-infra)

Рабочая GitOps-конфигурация домашнего k3s-кластера. Argo CD синхронизирует инфраструктурные приложения из Git; в репозитории описаны monitoring stack, cert-manager, Sealed Secrets и конфигурация самого Argo CD.

### [knowledge](https://github.com/AlexandrShapkin/knowledge)

Открытая база технических заметок на Quartz 5 и Obsidian. Для проекта настроены GitHub Actions, публикация через GitHub Pages, автоматическая проверка структуры и ссылок, безопасная синхронизация репозитория и миграционные утилиты.

Сайт: [knowledge.alexandrshapkin.ru](https://knowledge.alexandrshapkin.ru/)

### [devops-lab](https://github.com/AlexandrShapkin/devops-lab)

Учебный инфраструктурный проект. Реализовано создание виртуальных машин в Proxmox через Terraform и cloud-init, генерация Ansible inventory и базовый bootstrap узлов. Дальнейшая часть проекта с HA Kubernetes и GitOps пока приостановлена.

### [monitor-test-task](https://github.com/AlexandrShapkin/monitor-test-task)

Решение тестового задания по мониторингу Linux-процесса: Bash, systemd service и timer, журналирование событий, установка и удаление через Makefile.

### [docker-bta-server](https://github.com/AlexandrShapkin/docker-bta-server)

Docker-образ для запуска игрового сервера с конфигурацией через переменные окружения, persistent storage и примером Docker Compose. Образ публикуется в Docker Hub.

## Практический опыт

В коммерческой работе сопровождаю производственную информационную систему, анализирую инциденты и данные в Oracle Database, выполняю SQL-запросы и автоматизирую повторяющиеся операции на Python и Bash.

Один из внутренних Python-инструментов сократил ежедневную операцию выгрузки данных по 40 объектам примерно с 1,5 часа до 5-6 минут.
